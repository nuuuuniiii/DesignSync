# DesignSync PRD 실행 체크리스트 (단계별)

## 1. 제품 방향성 정립
- [ ] PRD `제품 개요(1)` 및 `목표 및 성공 지표(2)` 전체 리뷰
- [ ] KPI(활성 프로젝트 수, 평균 피드백 수, 응답 완료율, 요약 화면 재방문율) 확정 및 대시보드 항목 설계
- [ ] 사용자 인터뷰/조사 계획 수립, 정성 목표(“피드백 정리가 수월해졌다” 70%) 검증 로드맵 정의

## 2. 사용자/문제 검증
- [ ] `대상 사용자 & 주요 페르소나(3)` 별 페인포인트 목록화
- [ ] 최소 각 페르소나 3명 이상과 인터뷰 진행, 검증 결과 기록
- [ ] 인사이트를 `범위 정의(4)`에 반영하여 문제 진술 업데이트

## 3. MVP 범위 확정 및 일정 설계
- [ ] `MVP 범위(5)` 기능 목록 → 에픽/스토리 티켓 분해
- [ ] 프로젝트 등록 → 피드백 수집 → 리포트 → 재평가 사이클 플로우 다이어그램 작성
- [ ] M1/M2/Beta 마일스톤 (`출시 계획 & 리스크(12)`)을 기반으로 하향식 일정 정의

## 4. 개발환경 & 협업 세팅
- [ ] 코드 리포지토리 생성, 브랜치 전략/PR 규칙/코드리뷰 흐름 명시
- [ ] 공통 런타임 버전, 패키지 매니저, 스크립트(`setup`, `lint`, `test`) 정의
- [ ] 환경변수(.env 템플릿)와 비밀 정보 저장소 구성
- [ ] CI/CD 초안 구축: 빌드 → 테스트 → 린트 → 배포 단계 및 자동화 스크립트
- [ ] 협업 툴(Figma, Notion, Slack) ↔ 이슈 트래커(Jira/GitHub Projects) 연동

## 5. 정보 구조 & UX 설계
- [ ] `사용자 여정 및 화면 요구사항(7)` 기준 IA(Information Architecture)와 유저 플로우 작성
- [ ] 홈 / 프로젝트 등록 / 피드백 작성 / 피드백 요약 & 리포트 / 버전 타임라인 와이어프레임 제작
- [ ] 1~7점 스케일, 피드백 질문 최대 5개 UX 검증 및 안내 문구 작성
- [ ] CSV 다운로드, 그래프 표현(레이더/바 차트) UX 프로토타이핑

## 6. 기능 요구사항 상세화 (BE/FE 공통)
- [ ] `기능 요구사항(8)` 기반 도메인별 API/화면/유즈케이스 매트릭스 작성
- [ ] 피드백 초대 링크 공유 및 재평가 요청 플로우 상세 설계
- [ ] 에러 케이스/엣지 케이스 목록화 및 처리 정책 수립

## 7. 프론트엔드 아키텍처
- [ ] SPA/MPA 결정, 라우팅 구조 및 접근 제어 설계
- [ ] 상태관리 전략(Redux/Zustand 등)과 API 통신 계층(캐싱, 에러 핸들링) 설계
- [ ] 디자인 시스템 정의: 색상, 타이포, 컴포넌트 규칙, 토큰 관리
- [ ] 컴포넌트 트리 및 스토리북 구성, 주요 컴포넌트 문서화
- [ ] 접근성(A11y) 체크리스트 작성 (폼 라벨, 그래프 설명, 키보드 내비)
- [ ] 프론트엔드 테스트 전략: 유닛, 스냅샷, E2E(Playwright/Cypress) 계획

## 8. 백엔드 아키텍처
- [ ] API 사양(Swagger/OpenAPI) 작성: 프로젝트/피드백/리포트/타임라인 엔드포인트
- [ ] 인증·권한 모델: 소유자/리뷰어 역할, 토큰/세션 전략 정의
- [ ] 파일 업로드(이미지) & 외부 링크 관리, S3 계열 스토리지 연동 설계
- [ ] 리포트 집계 로직(평균/최대/최소, 버전 비교) 및 캐싱 전략 수립
- [ ] 로그/모니터링(Sentry 등), 알림(향후 확장) 고려한 아키텍처 문서화
- [ ] 백엔드 테스트 계획: 유닛, 통합, 컨트랙트 테스트 및 커버리지 목표

## 9. 데이터베이스 & 모델링
- [ ] `데이터 모델 초안(9)` 기반 ERD 작성 및 테이블 스키마 확정
- [ ] 피드백 유형별 점수(rating_map), focus_types 배열 구조 명세
- [ ] 인덱스 전략, 정합성(외래키/트랜잭션) 정책, 소프트 삭제 여부 결정
- [ ] 마이그레이션 도구 선택 및 초기 시드 데이터(더미 프로젝트/피드백) 정의
- [ ] 데이터 백업/복구 및 환경별(DB dev/staging/prod) 접속 정책 수립

## 10. 데이터 연결 & 통합 플로우
- [ ] 백엔드 ↔ DB 연결 설정 (ORM/쿼리 빌더, 연결 풀링) 및 환경별 설정 검증
- [ ] 백엔드 ↔ 프론트엔드 API 계약서(타입 정의, 응답 스키마) 확정
- [ ] 파일 스토리지, 외부 링크(예: Figma) 접근 권한 및 보안 정책 설정
- [ ] 배포 환경(스테이징/프로덕션)에서의 통합 테스트 체크리스트 작성

## 11. 비기능 요구사항 이행
- [ ] `비기능 요구사항(10)`에 따른 성능 목표(p95 3초, 저장 1초 이내) 측정 지표 설정
- [ ] 반응형/접근성 QA 플랜 수립, 자동화 가능 여부 검토
- [ ] 보안(세션 만료 12시간, 권한 체크) 테스트 케이스 정의
- [ ] 파일 업로드 한도(20MB), 예외 처리/알림 UX 구현 계획

## 12. 분석 및 추적
- [ ] `분석 및 추적(11)` 기반 이벤트 스키마 정의 (프로젝트 생성, 피드백 제출, 리포트 다운로드, 버전 생성 등)
- [ ] Amplitude/GA4 연동 설계, 태깅 명 명 규칙 및 발송 시점 결정
- [ ] 향후 히트맵 도구 도입 시나리오 및 데이터 프라이버시 검토

## 13. QA & 출시 준비
- [ ] 기능 테스트, 회귀 테스트, UAT(User Acceptance Test) 계획 수립
- [ ] 리뷰어 참여 저조 리스크 대응(리마인드 메시지, 알림) 실행 플랜
- [ ] 다양한 파일 포맷 요구/데이터 시각화 복잡도 리스크 완화 방안 검토
- [ ] 베타(내부 10팀) 운영 플랜: 초대, 온보딩 자료, 피드백 루프 정의

## 14. 확장 로드맵 기획
- [ ] `향후 확장 제안(13)` 항목별(자동 요약, 워크스페이스 권한, 외부 리뷰어 매칭, 모바일 앱) 선행 조사 범위 정의
- [ ] 출시 후 성공 지표 달성 기준과 Go/No-Go 판단 프레임워크 설정
- [ ] 커뮤니티/마켓플레이스, 워크플로 자동화 등 확장 기능 우선순위 재평가 주기 결정
