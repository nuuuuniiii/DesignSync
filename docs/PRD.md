# DesignSync PRD

## 1. 제품 개요
DesignSync는 디자이너가 프로젝트별 시안을 공유하고, 구조화된 피드백을 빠르게 수집·정리할 수 있는 웹 애플리케이션이다. 카테고리 기반 큐레이션과 정량/정성 피드백을 동시에 관리하여 프로젝트 의사결정을 지원한다.

## 2. 목표 및 성공 지표
- **1차 목표**: 프로젝트 등록 후 48시간 이내에 최소 3건의 피드백 확보 비율 60% 이상
- **정성 목표**: 사용자 인터뷰에서 “피드백 정리가 수월해졌다”는 응답 70% 이상
- **제품 활성 지표 (MVP)**
  - 활성 프로젝트 수 (최근 30일)
  - 프로젝트당 평균 피드백 수
  - 피드백 응답 완료율 (요청 대비 응답)
  - 피드백 요약 화면 재방문율

## 3. 대상 사용자 & 주요 페르소나
- **인하우스 디자이너**: 사내 프로젝트 리뷰를 정기적으로 진행하며, 다양한 직군으로부터 피드백을 수집해야 하는 사용자
- **프리랜서/에이전시 디자이너**: 외부 이해관계자에게 시안을 공유하고 투명한 피드백 히스토리를 남기고 싶은 사용자
- **리드 디자이너/PM**: 팀의 피드백 진행 현황을 한 곳에서 파악하고, 개선 후 결과를 추적해야 하는 의사결정자

## 4. 범위 정의
- **문제 범위**: 여러 채널(이메일, 메신저, Figma 댓글 등)에 흩어진 피드백을 모으기 어렵고, 개선 사항 추적·재평가가 비체계적이다.
- **해결 전략**: 프로젝트 단위의 구조화된 피드백 수집 → 카테고리별 자동 요약 → 개선 후 재평가 사이클 지원

## 5. MVP 범위
- 프로젝트 등록 (카테고리, 시안 업로드, 피드백 요청 질문, 기한 설정)
- 피드백 작성 (유형 선택, 1-7점 척도, 텍스트)
- 피드백 요약 & 리포트 (유형별 평균 점수, 그래프)
- 버전별 피드백 타임라인 (피드백 링크, 개선 후 재평가 요청)
- 기본 권한 모델: 프로젝트 소유자 + 초대한 리뷰어

## 6. 향후 확장 범위 (우선순위 순)
1. **피드백 큐레이션 고도화**: AI 기반 요약, 개선 제안 추천
2. **워크플로 자동화**: 슬랙/이메일 알림, 캘린더 연동, 피드백 리마인드
3. **성과 측정 대시보드**: 프로젝트별 KPI 연결, 기간별 트렌드 분석
4. **협업 기능**: 다중 파일 버전 관리, 토론 스레드, 멘션 기능
5. **마켓플레이스/커뮤니티**: 외부 리뷰어 매칭, 공개 포트폴리오 공유

## 7. 사용자 여정 및 화면 요구사항 (MVP)
### 7.1 Explore 화면
- 상단 GNB(Global Navigation Bar)에서 **Web/Apps** 선택 가능
- 카테고리별 필터링: 선택한 카테고리에 따라 해당하는 프로젝트만 표시
- 상태 필터: **Unsolved** / **Solved**로 프로젝트 분류하여 조회
- 프로젝트 목록: 카테고리와 상태에 맞는 프로젝트 카드 표시
- **Add New Project** 버튼: 클릭 시 프로젝트 등록 화면으로 이동

### 7.2 My Page 화면 - My Project 섹션
- 내가 등록한 프로젝트 목록 표시
- 프로젝트별 상태 표시: 새로운 피드백이 생성되면 해당 프로젝트에 **코멘트 아이콘** 표시
- **Select** 버튼: 프로젝트 선택 모드 활성화
- 선택 모드에서 선택한 프로젝트에 대해 다음 액션 수행 가능:
  - **Resolved**: 프로젝트를 해결됨 상태로 변경
  - **Edit**: 프로젝트 수정 화면으로 이동
  - **Delete**: 프로젝트 삭제

### 7.3 My Page 화면 - My Feedback 섹션
- 내가 피드백을 남긴 프로젝트 목록 표시
- 프로젝트별로 내가 작성한 피드백 요약 정보 표시

### 7.4 My Project 등록 화면
- **프로젝트 이름**: 텍스트 입력 필드
- **설명**: 프로젝트 설명 텍스트 입력 필드
- **App/Web 여부**: 라디오 버튼 또는 토글으로 선택
- **카테고리 선택**: 드롭다운 또는 선택 UI로 카테고리 지정
- **별점 평가 피드백 타입 선택**: 체크박스 형태로 평가할 피드백 타입 선택 (다중 선택 가능)
- **디자인 이미지 등록**: 이미지 업로드 기능 (드래그 앤 드롭 또는 파일 선택)
- **피드백 이름 선택 및 직접입력**: 
  - 선택지에서 피드백 이름 선택 가능
  - 직접 입력 옵션 제공

### 7.5 New Feedback > Rating 화면
- 슬라이더 UI로 1-5점 사이의 별점 조절 가능
- 슬라이더 조절 시 실시간으로 별점 표시 (별 아이콘 또는 숫자)

### 7.6 New Feedback > Screen 화면
- **Design 선택**: 등록되어 있는 Designs 중 하나 선택
- **Screen 선택**: 선택한 Design에 등록되어 있는 화면 목록 중 선택
- 선택한 화면에 대해:
  - 해당 화면에 요청된 피드백 질문 목록 표시
  - 각 질문에 대한 텍스트 입력 필드 제공

### 7.7 Project Overview 화면
- **별점 정보**: 프로젝트 전체 평균 별점 표시
- **Designs별 화면 및 피드백**:
  - Design 단위로 그룹화하여 표시
  - 각 Design의 화면 목록 및 해당 화면에 대한 피드백 요약 표시
  - 화면별 피드백 상세 조회 가능

## 8. 기능 요구사항 (MVP)
- **프로젝트 관리**
  - CRUD: 프로젝트 생성/수정/종료
  - 카테고리/기한 기반 필터링 및 정렬
- **시안 관리**
  - 이미지 업로드 (슬라이드형 미리보기)
  - 외부 링크(예: Figma) 저장 및 클릭 추적
- **피드백 수집**
  - 유형별 점수 저장 구조 (프로젝트ID, 버전ID, 리뷰어ID, 유형, 점수, 코멘트)
  - 텍스트 피드백 저장 및 검색
  - 피드백 초대 링크/이메일 발송 (MVP는 URL 공유, 이메일은 확장 기능)
- **리포팅**
  - 평균/최대/최소 점수 계산
  - 버전별 비교 그래프
  - CSV 내보내기

## 9. 데이터 모델 초안 (MVP)
- `Project`: id, name, category, background, goal, due_date, owner_id, status, created_at
- `ProjectVersion`: id, project_id, version_label, description, assets (이미지·링크 메타), created_at
- `Feedback`: id, project_version_id, reviewer_id, focus_types[], rating_map{type:score}, comment_text, created_at
- `FeedbackQuestion`: id, project_id, content, order
- `User`: id, name, email, role

## 10. 비기능 요구사항
- 반응형 웹 (데스크톱 우선, 태블릿 대응)
- 파일 업로드: 20MB 이하 이미지(JPG/PNG), 확장성 고려한 S3 계열 스토리지 가정
- 접근성: 최소 AA 대비 대비도, 키보드 사용 가능
- 보안: 로그인 세션 만료 12시간, 프로젝트 접근 권한 체크
- 성능: 주요 화면 로딩 3초 이내 (p95), 피드백 저장 1초 이내 응답

## 11. 분석 및 추적 (MVP)
- 이벤트: 프로젝트 생성, 시안 업로드, 피드백 제출, 리포트 다운로드, 버전 생성
- 파이프라인: Amplitude/GA4 연동 가정, 향후 히트맵 도구 연동 고려

## 12. 출시 계획 & 리스크
- **M1 (4주)**: 프로젝트 등록 & 피드백 수집 플로우 구축
- **M2 (4주)**: 리포트 뷰 + 타임라인
- **베타**: 내부 디자이너 10팀 대상 피드백 수집
- **리스크**: 리뷰어 참여 저조 → 알림/리마인드 기능 필요, 다양한 파일 포맷 요구 → MVP 범위 벗어남, 데이터 시각화 복잡도 → 단순 그래프부터 시작

## 13. 향후 확장 제안 요약
- 자동 요약 및 인사이트 추천 (AI)
- 팀 단위 권한/워크스페이스 관리
- 외부 리뷰어 매칭 플랫폼 연계
- 모바일 앱 (리뷰 작성 전용)
